# Prioritize daily use over portfolio polish

The project brief (`closetos_project_and_infrastructure.md`) pitches ClosetOS as a portfolio showcase of GitHub Agentic Workflows, serverless infra, and self-improving agents. We are instead building it **daily-use first**: every design choice optimizes for a tool the user will actually reach for to get dressed, and the portfolio-impressive machinery is layered on only after the core earns trust. When the two goals conflict, **daily use wins the tie**.

_Why it matters:_ a wardrobe assistant that gives bad advice is an unimpressive portfolio piece regardless of the plumbing, and the self-improvement story is only credible once real usage generates real feedback.
